---
title: Using a Local Jextract Installation
description: How to use a locally installed version of Jextract with the Gradle Jextract plugin
sidebar:
  order: 4
  label: Local Installation
---

Setting the Gradle property `org.openjdk.jextract.installation-path` to a directory path tells the plugin to use a local Jextract version.
This can be useful to avoid network requests or for specific development/testing scenarios.

```properties {2}
# gradle.properties
org.openjdk.jextract.installation-path=/usr/local/opt/jextract-22/
```

:::tip
Relative paths are resolved from the project directory.
:::

Alternately, a local installation directory can be configured in the dSL extension.

<Tabs syncKey="dsl">
<TabItem label="Kotlin">
```kts {3}
// build.gradle.kts
jextract {
    installation.location = layout.projectDirectory.dir("/home/user/bin/jextract-25/")
}
```
</TabItem>
<TabItem label="Groovy">
```groovy {3}"
// build.gradle
jextract {
    installation.location = layout.projectDirectory.dir '/home/user/bin/jextract-25/'
}
```
</TabItem>
</Tabs>

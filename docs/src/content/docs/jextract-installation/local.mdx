## Use a Local Installation

You can explicitly set the path to a local Jextract installation in the plugin's DSL.
In this case, the plugin will ignore any download-related configuration.

:::caution
This method meets an offline-requirement, but your build becomes dependent of your local setup.
:::

<Tabs syncKey="dsl">
<TabItem label="Kotlin">

```kotlin gradle dsl "local"
// build.gradle.kts
jextract.generator {
    local = layout.projectDirectory.dir("/usr/local/opt/jextract-22/")
}
```

</TabItem>
<TabItem label="Groovy">

```groovy gradle dsl "local"
// build.gradle
jextract.generator {
    local = layout.projectDirectory.dir "/usr/local/opt/jextract-22/"
}
```

</TabItem>
</Tabs>

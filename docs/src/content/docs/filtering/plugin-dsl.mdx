This plugin offers filtering without arg files through DSL properties where the symbols can be added.
This approach might be more beginner-friendly or facilitate exploration of available options.

<Tabs syncKey="dsl">
<TabItem label="Kotlin">

```kts "typedefs" "functions" "constants"
// build.gradle.kts
sourceSets.main {
  jextract.libraries.register("<libname>") {
    whitelist {
      typedefs.add("MY_TYPE")
      functions.add("MY_FN")
      constants.add("MY_CONST")
      // more categories
    }
  }
}
```

</TabItem>
<TabItem label="Groovy">

```groovy "typedefs" "functions" "constants"
// build.gradle
sourceSets.main {
  jextract.libraries.register("<libname>") {
    whitelist {
      typedefs += "MY_TYPE"
      functions += "MY_FN"
      constants += "MY_CONST"
      // more categories
    }
  }
}
```

</TabItem>
</Tabs>

The distinct symbol categories like typedefs, constants, or functions have their separate list properties in the `jextract.whitelist` extension.
